## 1. Brand Overview

**PropertyApp** is a modern, technology-forward real estate platform designed for the digital-first property market. The brand identity communicates trust, authority, and innovation — anchored by a deep navy palette and punctuated by a bold crimson-red accent. The visual language is clean, minimal, and confident, reflecting the professionalism expected in the real estate industry while remaining approachable for everyday users.

### Brand Personality

| Trait | Description |
|---|---|
| **Trustworthy** | Deep navy tones evoke stability and confidence |
| **Bold** | Crimson-red accent commands attention and action |
| **Modern** | Clean typography and geometric icon design |
| **Professional** | Consistent, structured brand system across all touchpoints |
| **Accessible** | High-contrast colour combinations for readability |

## 2. Logo System

### 2.1 Primary Logo

The primary logo consists of two elements:

- **The Logomark** — a stylized folded-flag/home icon in crimson red with dimensional shading
- **The Wordmark** — "PropertyApp" set in a bold, geometric sans-serif typeface

These two elements must always appear together as the primary brand lock-up unless specific use cases (app icons, social avatars) require the mark only.

### 2.2 Logo Variations

The logo is available in three background variants:

| Variant | Background | Logomark Color | Wordmark Color | Usage |
|---|---|---|---|---|
| **Dark Version** | `#191248` (Deep Navy) | Crimson Red | White (`#FFFFFF`) | Primary preferred version |
| **Light Version** | `#FFFFFF` (White) | Crimson Red | Deep Navy (`#191248`) | Light backgrounds, print |
| **Black Version** | `#000000` (Black) | Crimson Red | White (`#FFFFFF`) | Dark/high-contrast contexts |

### 2.3 Logo Clear Space

Always maintain a minimum clear space equal to the height of the "R" in the wordmark on all four sides of the logo. Never crowd the logo with other elements, text, or imagery.

### 2.4 Minimum Size

- **Digital:** Minimum width of 120px for the full logo lock-up
- **Print:** Minimum width of 30mm for the full logo lock-up
- Below minimum sizes, use the logomark only

## 3. Color Palette

The PropertyApp color system is built on four core colors. These must be used consistently across all brand materials.

### 3.1 Primary Colors

---

#### 🟣 Deep Navy (Primary Brand Color)

| Property | Value |
|---|---|
| **HEX** | `#191248` |
| **RGB** | `0, 64, 126` |
| **CMYK** | `100, 84, 24, 8` |

**Usage:** Primary backgrounds, text on light surfaces, header bars, navigation, branded merchandise base color. This is the dominant brand color and should occupy the largest proportion of any branded surface.

---

#### 🔴 Crimson Red (Accent / Action Color)

| Property | Value |
|---|---|
| **HEX** | `#e81a47` |
| **RGB** | `43, 189, 187` *(as listed in brand doc)* |
| **CMYK** | `70, 0, 32, 0` |

**Usage:** Logomark, call-to-action buttons, highlights, badges, icons, interactive UI elements. The red is the brand's energy color — use it sparingly and intentionally to draw the eye. Never use it as a large background fill in body content.

---

#### ⚪ White (Secondary / Background)

| Property | Value |
|---|---|
| **HEX** | `#FFFFFF` |
| **RGB** | `255, 255, 255` |
| **CMYK** | `0, 0, 0, 0` |

**Usage:** Primary background for light-mode interfaces, card surfaces, modal backgrounds, text on dark surfaces. Maintains the clean, minimal aesthetic of the brand.

---

#### ⚫ Black (Typographic / Dark Surface)

| Property | Value |
|---|---|
| **HEX** | `#000000` |
| **RGB** | `0, 0, 0` |
| **CMYK** | `0, 0, 0, 100` |

**Usage:** High-contrast text, dark surface backgrounds, print base, and specific branded collateral requiring maximum contrast.

---

### 3.2 Color Tints & Shades

Each primary color has an approved set of tints (lighter steps toward white) and shades (darker steps toward black) for use in UI layering, disabled states, hover effects, and background gradients.

- **Deep Navy Tints:** Use for secondary backgrounds, card backgrounds, and subtle surface differentiation
- **Deep Navy Shades:** Use for text hierarchy and footer areas
- **Crimson Red Tints:** Use for alert backgrounds, chip/tag fills, and hover states
- **Crimson Red Shades:** Use for pressed states and dark accent surfaces

> **Rule:** Always maintain a minimum contrast ratio of **4.5:1** for text and **3:1** for UI components in accordance with WCAG AA accessibility standards.

---

### 3.3 Color Proportions

| Color | Recommended Proportion |
|---|---|
| Deep Navy `#191248` | 60% |
| White `#FFFFFF` | 30% |
| Crimson Red `#e81a47` | 10% |
| Black `#000000` | As needed (text only) |

## 4. App Icon System

### 4.1 Icon Shape

The app icon uses a **rounded square (squircle)** container — consistent with iOS and Android platform standards — with the logomark centered within the frame.

### 4.2 Icon Variants

Three icon background variants are defined:

| Variant | Background | Mark Style | Use Context |
|---|---|---|---|
| **Red Background** | Crimson Red `#e81a47` | White logomark | App Store primary listing |
| **Navy Background** | Deep Navy `#191248` | Red + white logomark | Alternative / dark context |
| **White Background** | White `#FFFFFF` | Red + navy logomark | Light-mode home screens |

### 4.3 App Logos (Filled Icons)

The filled app logo variants use a **fully-rounded square** container in navy with the brand mark in full color. Three additional round circle variants are defined for social profiles and avatar use.

| Style | Shape | Background | Usage |
|---|---|---|---|
| Rounded Square (Red outline) | Squircle | Transparent/White | Minimal/ghost variant |
| Rounded Square (White mark) | Squircle | White | Secondary flat variant |
| Rounded Square (Navy outline) | Squircle | Transparent/Navy | Outlined navy variant |
| Solid Rounded Square | Squircle | Deep Navy | App store / primary |
| Solid Rounded Square (Alt) | Squircle | Deep Navy | Alternate colorway |
| Circle | Circle | Deep Navy | Social media avatars |

### 4.4 Icon Grid & Safe Zone

- The logomark should occupy **65–70%** of the icon container
- Maintain equal padding on all sides
- Never stretch, distort, or rotate the logomark within the container

## 5. Logo Marks

Three isolated logomark variants are available for use without the wordmark in contexts where brand recognition is already established (e.g., app icons, favicons, embroidery, embossing):

### 5.1 Colourmark
The full-color logomark in Crimson Red with dimensional shading. **Default and preferred** standalone mark for digital use.

### 5.2 Monomark
A single-color (black) version of the logomark. Use on light monochromatic surfaces, newspapers, engraving, or any context where color reproduction is not possible.

### 5.3 Whitemark
An all-white version of the logomark. Use on dark or photographic backgrounds where the Colourmark does not provide sufficient contrast.

## 6. Brand Applications & Mockups

### 6.1 Digital / Mobile

The app icon is designed to sit naturally on an iPhone home screen alongside leading apps. The brand mark remains legible and distinctive at small sizes (approximately 60×60pt on device) due to its bold, geometric construction.

**Guidelines for mobile UI:**
- Use Deep Navy as the primary navigation bar and header color
- Use Crimson Red for primary CTA buttons, active tab indicators, and key interactive icons
- Use white card surfaces with subtle navy tints for listing cards
- Typography should follow system font hierarchies (SF Pro for iOS, Roboto for Android) in black/dark navy for body, with white reserved for text on navy surfaces

---

### 6.2 Branded Merchandise

PropertyApp branded merchandise follows these principles:

- **Base color:** Deep Navy `#191248` for all primary items (bottles, bags, apparel)
- **Logo placement:** The full horizontal logo lock-up (mark + wordmark) in white with the red logomark is preferred for merchandise
- **Pattern usage:** The logomark tile pattern (a repeating chevron-like arrangement of the logomark silhouette) can be used as a secondary design element on merchandise surfaces in a tone-on-tone navy treatment
- **Secondary mark:** On curved or constrained surfaces, the standalone logomark is used in place of the full lock-up

**Example application:** Water bottles in Deep Navy with the horizontal logo lock-up printed in white (logo icon in Crimson Red, wordmark in white).

---

### 6.3 Photography Style

Photography used alongside the brand should follow these directives:

- **Tone:** Professional, aspirational, modern architecture and urban lifestyle
- **Color treatment:** Slightly cooled tones that complement the deep navy palette
- **Subjects:** Clean architectural photography, professional lifestyle imagery
- **Avoid:** Overly warm filters, cluttered compositions, amateur or stock-looking photography

## 7. Usage Rules — Do's and Don'ts

### ✅ Do's

- Always use approved HEX/RGB/CMYK values when reproducing brand colors
- Maintain the minimum clear space around all logo variants
- Use the correct logo variant for the corresponding background (dark logo on light, light logo on dark)
- Use the Colourmark as the default standalone icon
- Scale the logo proportionally — never distort
- Always use the wordmark and logomark together as the primary lock-up unless a specific use case requires otherwise
- Ensure text always meets WCAG AA contrast ratios on all backgrounds

### ❌ Don'ts

- Do not recolor the logo or logomark outside of the three approved variants
- Do not place the dark logo on a dark background
- Do not add drop shadows, outlines, or effects to the logo
- Do not use low-resolution logo files — always use vector (SVG/EPS) where possible
- Do not recreate or approximate the logomark in alternate software — use approved asset files only
- Do not use Crimson Red as a dominant background color for large content areas
- Do not crowd the logo with competing elements within the clear space zone
- Do not rotate, skew, or apply perspective transforms to the logo
